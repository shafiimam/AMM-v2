## What This Is

A MA MANIÉRE (AMM) Shopify theme — a heavily customized version of **Prestige by Maestrooo v10.4.0**. It is a Shopify Online Store 2.0 theme. Upstream docs: https://support.maestrooo.com/category/749-technical-documentation

## Development Commands

```bash
# Authenticate with Shopify (one-time)
shopify auth login

# Push theme to store / start dev server with hot-reload
shopify theme dev --store=<store-handle>

# Push all files to store
shopify theme push

# Pull latest theme from store
shopify theme pull

# Check theme for errors
shopify theme check
```

No build step — all assets (`theme.js`, `theme.css`, `vendor.min.js`) are pre-compiled and committed directly to `assets/`.

## Architecture Overview

### Template System (OS 2.0)
Templates in `templates/` are JSON files that wire sections together. Many are brand/campaign-specific variants:
- `collection.[brand].json` — per-brand collection pages (adidas, loewe, jacquemus, etc.)
- `product.[product].json` — per-product pages (ray-ban, eql-launches, pre-order, etc.)
- `article.[campaign].json` — editorial article layouts per campaign (nike-yardrunners, resort-edit-blog, etc.)

The default fallbacks are `collection.json`, `product.json`, `article.json`.

### Section Groups
- `sections/header-group.json` — sticky header, announcement bar, nav
- `sections/footer-group.json` — footer + text-with-icons
- `sections/overlay-group.json` — newsletter popup, cart drawer

These are rendered via `{%- sections 'header-group' -%}` in `layout/theme.liquid`.

### JS Architecture
Uses native ES modules with an **importmap** (polyfilled for Safari ≤16.3 via `es-module-shims.min.js`):
- `theme.js` — main theme module
- `vendor.min.js` — third-party bundle
- `jquery.min.js`, `angle.js`, `360-knob.js` — loaded separately via `defer`

Global config is exposed at `window.themeVariables` (see `snippets/js-variables.liquid`).

### CSS Architecture
- `assets/theme.css` — base Prestige theme styles (do not edit; comes from upstream)
- `assets/custom.css` — all AMM customizations go here; loaded after theme.css
- `snippets/css-variables.liquid` — CSS custom properties for typography, spacing, color schemes
- Responsive breakpoints: `sm` = 700px, `md` = 1000px, `lg` = 1150px

### Key Snippets
- `snippets/css-variables.liquid` — defines all `:root` CSS vars from theme settings
- `snippets/js-variables.liquid` — exposes `window.themeVariables`
- `snippets/product-gallery.liquid` — reused in `main-product.liquid`
- `snippets/locksmith*.liquid` — Locksmith app integration for access control

### Third-Party Integrations
- **Google Tag Manager** — GTM-M2X2R69X, hardcoded in `layout/theme.liquid`
- **Locksmith** — wraps `layout/theme.liquid` head/body for access gating
- **PhotoSwipe** — imported via importmap as `"photoswipe"`

### PWA
`assets/manifest.json` and `assets/service-worker.js` implement PWA support. IndexedDB (`amm-pwa-db`) is initialized in `layout/theme.liquid`.

## Conventions

- Custom styles belong in `assets/custom.css`, never in `assets/theme.css`.
- Brand/campaign-specific layouts get their own template file (`collection.[brand].json`) rather than modifying the default.
- `templates/*.json` files are auto-generated by Shopify's theme editor — comments at the top of each warn against manual edits; changes can be overwritten by the editor.
- `sections/footer-group.json` and `sections/header-group.json` are also auto-generated.
- Localization strings live in `locales/en.default.json`; schema labels use the `t:` prefix referencing `locales/en.default.schema.json`.
